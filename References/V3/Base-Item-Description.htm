by Dr. Zed

Note, any previous changes you have made may conflict with these changes. Please make sure your data is identical to the 'Was' state before changing it to the 'Now' state.

00080810 xxxx xxxx xxxx xxxx 5761 7272 696F 7227 ....amerWarrior'

00080820 7320 6669 7273 7420 7377 6F72 6400 0000 s First Sword...

Each base item which is to gain a description line needs to be listed. The first byte is the base item number. 01 is the Warrior's starting sword (used as an example). The following four bytes (in reverse order) are a pointer to the description to be used. If the above data area is used for the text, add 00401600 to the file offset to get the pointer. For the example, 181E4800 points to the text at 00080818 (00080818 + 00401600 = 00481E18; reversed becomes 181E4800).

The list must be sortedby item number, from 80 to FF, then from 0 to 7F. There must be an entry for 7F, which is currently assigned a null string (7F, 301E4800). The pointer to the null string could be replaced with a pointer to any text desired for that item number, if required.
